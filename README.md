# Kontur ETRN Reconstruction

## Назначение пакета

Это публичный overlay-срез реконструкционного слоя legacy-адаптера Контур ЭТрН.

Пакет собран из актуальных исходников `7. Реконструкция\Исходники` и имеет файловую структуру, согласованную с корнем проекта `tis`.

## Что входит

- `Account` — WebForms UI и composition root;
- `Generated` — модели, storage, gateways и use case;
- `Modules` — legacy-модули ЭТрН, используемые как совместимость;
- `Sql` — SQL-границы реконструкции;
- `Документация` — архитектурный и рабочий контекст без raw-log приложений;
- `tis.csproj` — обновленный проектный файл для наложения на базовый проект;
- `PROCESS_NOTE.md` — правила процесса и архитектурные ограничения пакета.

## Что сознательно не включено

Из публичного пакета удалены:

- raw-log конкретных прогонов;
- приватная переписка и рабочие заметки;
- операционные артефакты по контрольному прогону;
- build-артефакты `bin` и `obj`.

## Примечание по документации

Часть исторических документов в папке `Документация` может ссылаться на внутренний контекст реконструкции, который хранится отдельно.

Это нормально для рабочего handoff-среза:

- публичный overlay-пакет содержит только код и неперсонализированную документацию;
- приватные raw-log, переписка и материалы конкретных прогонов лежат только в соседней папке `private_context`.

## Как использовать

1. Взять базовый legacy-проект `tis`.
2. Наложить содержимое этого пакета поверх корня проекта.
3. Проверить наличие внешних legacy-зависимостей проекта в `bin` рабочего контура.
4. Собрать проект в привычном окружении .NET Framework 4.0 / WebForms.

## Ограничение

Пакет не является полной копией всего legacy-проекта и не заменяет основной контур целиком.

Его цель — передать:

- актуальную структуру реконструкционного слоя;
- текущую архитектурную модель;
- совместимый `tis.csproj`;
- минимально достаточную документацию для продолжения работы.
