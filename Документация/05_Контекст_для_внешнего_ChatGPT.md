# Контекст для внешнего ChatGPT по реконструкции Контур ЭТрН

## Назначение документа

Этот документ нужен как автономный пакет контекста для внешнего ChatGPT, который не имеет доступа:

- к локальному диску разработчика;
- к исходникам проекта на ПК;
- к базе данных;
- к текущему состоянию рабочей папки Codex.

Документ должен позволять внешнему ChatGPT продолжить архитектурный разбор и давать осмысленные рекомендации без прямого доступа к окружению.

  - `C:\Users\gl\source\repos\Kontur\1. Текущий проект`
- реконструкционный слой:

Все новые архитектурные изменения, описанные ниже, выполнялись в `7. Реконструкция`.

Основной проект нельзя ломать. Он сохраняется как эталон для сравнения.

## Технологический контекст

Стек проекта:

- ASP.NET WebForms
- .NET Framework 4.0
- C#
- MSSQL

Особенности:

- система legacy;
- значимая часть бизнес-логики живет в SQL;
- нельзя мыслить как в новом .NET-проекте;
- изменения должны быть локальными и аккуратными;
- бизнес-логика не должна расползаться по UI;
- ad-hoc SQL в code-behind недопустим;
- целевая модель: Ports & Adapters.

## Исходная постановка реконструкции

Нужно не чинить старый адаптер точечно, а реконструировать архитектуру адаптера Контур ЭТрН на отдельном слое, сохраняя основной проект нетронутым.

Ключевые выводы, зафиксированные до начала текущих правок:

- текущая реализация считает `T1` завершенным слишком рано;
- `T1_INITIAL`, вероятно, создает документ/черновик, но не завершает полноценную отправку подписанного первого титула;
- `T2` нельзя моделировать как еще одну почти такую же отправку;
- правильная модель процесса:
  - `T1` = first title;
  - `T2/T3/T4` = recipient/response titles;
- UI должен стать тонким слоем;
- бизнес-процесс должен быть вынесен в use case;
- состояние этапа должно стать явной моделью.

## Зафиксированный прикладной дефект

Во время ручного прогона был получен конкретный отказ Контур:

`MessageToPost.DocumentAttachments[0]: { Location = /Файл/Документ/СодИнфГО/СвВодит/@ИННФЛ, Message = ИНН 123456789012 не удовлетворяет контрольному соотношению }`

### Что удалось локализовать

Источник был не во внешнем API и не в сертификате.

Источник был внутри реконструкционного слоя:

- `Generated\Kontur\Integration\KonturT1ParticipantOverrideService.cs`

Там существовала тестовая подстановка:

- `СвВодит/@ИННФЛ = 123456789012`

### Что уже исправлено

Исправление сделано только в `7. Реконструкция`:

- фиктивный `ИННФЛ` больше не подставляется;
- если ИНН водителя невалиден, он очищается;
- fallback теперь доводит `СвВодит` только через реквизиты водительского удостоверения;
- добавлена локальная проверка корректности ИНН физлица перед сохранением.

### Важное следствие

Старые уже собранные XML не исправятся автоматически.

После этой правки для проверки нужно:

1. заново пересобрать `T1`;
2. заново подписать новый XML;
3. заново импортировать `.sgn`;
4. только потом повторять отправку.

## Краткая сводка в одном абзаце

В проекте идет реконструкция адаптера Контур ЭТрН на отдельном слое `7. Реконструкция` без затрагивания основного проекта. Уже введены явное состояние этапа, repository, screen-model, transition-policy, gateway-порты и send/build/signature use case-ы, а `KonturProbe` подключается к ним только через add-only bridge с fallback на legacy runtime. Главный незакрытый риск связан с правильной семантикой `T1`, а из свежих практических проблем уже локализован и исправлен дефект с невалидным `ИННФЛ` водителя (`123456789012`) в нормализаторе `T1`. Внешнему ChatGPT нужно исходить из того, что основной проект трогать нельзя, новый runtime нельзя включать резко, а следующий фокус должен быть на корректном завершении `T1` и безопасном истончении UI.
