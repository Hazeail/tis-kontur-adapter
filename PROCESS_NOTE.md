# Процесс и ограничения

## Технологическая рамка

- ASP.NET WebForms
- .NET Framework 4.0
- C#
- MSSQL

## Обязательные ограничения

- работать локально и минимально;
- не предлагать новые платформенные фичи, которые не лягут в основной контур;
- использовать подход Ports & Adapters;
- не переносить бизнес-логику в UI;
- не писать ad-hoc SQL в code-behind;
- использовать явные use case для бизнес-сценариев;
- сохранять совместимость с legacy-контуром, если это не ломает реконструкцию.

## Смысл текущего слоя

Реконструкция ведется отдельно от основного проекта, чтобы:

- сохранить рабочий legacy-контур нетронутым для сравнения;
- вынести процесс ЭТрН из UI в use case-слой;
- сделать состояние этапа явной моделью;
- не считать `T1_INITIAL` завершением T1 слишком рано.

## Текущий архитектурный фокус

- `T1` рассматривается как first title;
- `T2/T3/T4` рассматриваются как recipient/response titles;
- `Completed` означает завершение роли ТИС по этапу, а не завершение всего документооборота Контур;
- UI должен работать как тонкий composition root.
